# Three-Month Scope

The goal for the next three months is to shape an early recommendation system for hedge fund managers and investment teams. At this stage, the product should stay open-ended enough that we can pivot based on what feels strongest, but the core direction is clear: the system should help users ask market, stock, portfolio, risk, and backtesting questions, and then return structured recommendations backed by data, agent analysis, risk checks, and clear reasoning.

In the first month, the focus should be on making current foundation strong and making the product shape clearer. We should define the kinds of questions the system should be able to answer and create a small benchmark set of our representative queries with broader expected answers. This benchmark set can become our reference point for checking whether the engine is improving over time. Along with that, the query routing schema and recommendation output schema should be standardized, so every query can be logged with its extracted parameters and every answer follows a consistent structure. By the end of the first month, the system should have a clear base flow: a user asks a question, the system captures the intent and context, maps it to a workflow, and produces a structured recommendation format.

In the second month, the focus should move toward recommendation intelligence. This is where the router output should start connecting to the right workflows, such as stock recommendation, portfolio review, risk analysis, backtesting, valuation, news sentiment, or asset comparison. The agents should also start producing stronger outputs, not just simple signals, but evidence, risks, confidence, reasoning, and what would change the recommendation. We can also start experimenting with a small BERT or SLM-style router during this phase. The goal is not to make the router perfect immediately, but to validate whether it can reliably classify the main query types and produce useful structured parameters for the rest of the system.

In the third month, the focus should be evaluation and a mvp. The benchmark query set should be used repeatedly to compare outputs across iterations, so we can see whether the recommendation engine is getting closer to the expected answers or drifting away. The backtesting flow should also become easier to use, so we can show how certain recommendations or strategies would have performed historically. By the end of the third month, the system should be able to demonstrate a complete product loop: a hedge fund manager asks a question, the system understands the request, gathers the relevant data, runs the research agents, applies risk and portfolio logic, and returns a structured recommendation with explanation, confidence, evidence, risks, and sizing.

The three-month outcome does not need to be a finished trading platform. The more realistic goal is to have a credible recommendation engine prototype that is structured, testable, explainable, and flexible enough to support multiple future directions. From there, the product can move deeper into stock recommendations, portfolio review, analyst workflow automation, risk intelligence, backtesting, or eventually execution support depending on where the strongest client value appears.
